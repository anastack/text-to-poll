## Quiz Poster Bot (Telegram)

Бот помогает пошагово собрать тест: тема, сообщение перед тестом, вопросы по одному, затем отправка в выбранный канал сразу или по таймеру.

После `/start` бот показывает inline-кнопки: создание теста, очередь отложенных тестов, смена канала и помощь. Быстрый старт конструктора также доступен командой `/test`.

### Требования

- Python 3.10+
- Бот добавлен в канал как админ с правом публикации сообщений и опросов.

### Быстрый старт (Windows / PowerShell)

```bash
py -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -r requirements.txt
copy quiz_poster_bot\.env.example quiz_poster_bot\.env
notepad quiz_poster_bot\.env
python -m quiz_poster_bot.quizbot.main
```

### Настройки

`BOT_TOKEN` обязателен.

`TARGET_CHANNEL_ID` можно указать в `.env`, например `-1001234567890` или `@your_public_channel`. Если оставить пустым, бот попросит канал в чате. Канал также можно заменить командой `/channel`.

### Создание теста

Нажмите `Создать тест` или отправьте `/test`.

Бот последовательно попросит:

1. Тему теста.
2. Сообщение перед тестом, например `Сейчас будет викторина по теме «циклы».`
3. Вопросы по одному.
4. Выбор: отправить сейчас или отложить.

### Формат вопроса

```text
Что выведет цикл?


*0 1 2
1 2 3
ошибка
```

Первая непустая строка считается вопросом. Между вопросом и вариантами ответов можно оставлять любое количество пустых строк и пробелов.

Правильный ответ обязательно помечается `*` или `+` в начале строки. Если не отметить правильный вариант, бот попросит отправить вопрос еще раз и пометить правильный ответ.

Если правильных вариантов несколько, бот отправит обычный опрос с множественным выбором.

### Фото

Вопрос можно отправить фото с подписью в формате вопроса. Также можно сначала отправить фото, а затем текст вопроса. Фото будет опубликовано перед соответствующим вопросом.

### Отложенная отправка

После добавления всех вопросов нажмите `Готово`, затем выберите `Отложить`.

Поддерживаются варианты кнопками и свой формат времени: `30m`, `1h`, `2h`, `1d`, `через час`, `через 2 часа`. Если указать просто число, оно считается минутами.

Команда `/scheduled` показывает текущую очередь отложенных тестов. Очередь хранится в `scheduled_quizzes.json`, поэтому не пропадает после перезапуска бота.
